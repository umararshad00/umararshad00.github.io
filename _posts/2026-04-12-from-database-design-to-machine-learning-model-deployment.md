## Expanding Our Project with a Larger Dataset

After completing the database design, normalization, and implementation phases of our project, we moved to the next stage, which was integrating Machine Learning into our work. This phase was particularly exciting because it allowed us to combine the knowledge we had gained from Database Systems with the Machine Learning concepts we had learned in the previous semester.

For this stage, we worked with a larger and more detailed dataset. Unlike our previous Machine Learning project from the first semester, where we had used a randomly selected dataset, this time we focused on a dataset that had been researched, organized, and prepared more carefully. The experience from the previous semester proved very useful because we already understood the basic workflow of machine learning projects, including data preparation, preprocessing, model training, and evaluation.

One important difference was that this time we paid greater attention to data quality and database structure before moving to model training. Since we had already applied database design principles and normalization techniques, the dataset was much more organized and suitable for analysis. This helped us appreciate the connection between database management and machine learning, as both fields rely heavily on clean and structured data.

## Training the Machine Learning Model

Once our dataset was ready, we moved on to the model training phase. This was a practical application of the concepts we had learned during our Machine Learning studies under the guidance of Dr. Bilal Ahmad. In the previous semester, we had explored machine learning algorithms and trained models using sample datasets. However, this project felt more meaningful because we were applying those concepts to a dataset that had been carefully researched and prepared.

Before training the model, we ensured that the data was properly organized and normalized. This preparation helped improve the quality of the results and reduced the chances of errors during model training. After preprocessing the data, we selected an appropriate machine learning model and began the training process.

Watching the model learn from the dataset and generate predictions was one of the most rewarding parts of the project. It allowed me to see how theoretical concepts are transformed into practical solutions. Through this experience, I gained a deeper understanding of the complete machine learning workflow and the importance of data preparation before training a model.

One lesson I learned during this phase was that the success of a machine learning model depends heavily on the quality of the data being used. A well-organized dataset can significantly improve model performance, while poorly structured data can lead to inaccurate results.

## Creating the Interface and Completing the Project

After successfully training the machine learning model, we focused on making the project more user-friendly. Instead of keeping the model as a simple script, we wanted to create an interface through which users could interact with the system easily.

To achieve this, we developed the necessary files required for deployment and connected the trained model to a web-based interface. We also worked with APIs to enable communication between different parts of the system. This stage introduced us to another important aspect of software development: transforming a trained machine learning model into a usable application.

After integrating the model, APIs, and interface components, our project was finally completed. Looking back, this was one of the most comprehensive academic projects I had worked on so far. It combined database management, normalization, data preprocessing, machine learning model training, and basic deployment concepts into a single project.
