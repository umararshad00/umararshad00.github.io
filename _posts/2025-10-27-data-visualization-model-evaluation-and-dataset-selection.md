## Learning to Read Data Through Visuals

During my university journey, one of the most valuable learning experiences was understanding how to work with data effectively through visualization, evaluation, and proper dataset selection. These skills played a crucial role in strengthening my knowledge of machine learning and helped me move from basic concepts to practical implementation. At the beginning of this phase, I focused on learning data visualization techniques, which are essential for interpreting and presenting data in a meaningful way. I studied different types of charts and graphs, including bar charts, histograms, box plots, bar plots, and scatter plots. Each of these visualizations has its own purpose; for example, bar charts help compare categories, histograms show data distribution, and scatter plots help identify relationships between variables. Learning how to create these visualizations through coding allowed me to better understand patterns, trends, and insights within datasets. This not only improved my analytical thinking but also helped me present data in a more clear and professional manner.

At first, I thought charts were mainly used to make a project look attractive, but this phase changed my opinion. I learned that visualization is actually a thinking tool. When data is shown visually, hidden patterns become easier to notice. A bar chart can quickly show which category is higher, a histogram can reveal whether values are spread evenly or concentrated in one range, and a scatter plot can show whether two variables may be connected. These details are difficult to understand from raw rows and columns alone. This made me realize that good visualization can guide the next steps of analysis.

Practicing visualization code also improved my confidence in Python. I had to remember syntax, choose the right chart type, label axes, and make the output understandable. Small details like titles, labels, and colors mattered because they made the chart easier to read. This taught me that technical work should be clear for both the developer and the audience. If I build a portfolio website, visitors should be able to understand what I did without needing to ask for every explanation. Visualization helps create that bridge between technical analysis and communication.

## Understanding Model Evaluation

After gaining a solid understanding of visualization, I moved on to learning how to evaluate machine learning models. Model evaluation is a critical step because it helps determine how well a model is performing. I studied different evaluation techniques such as the Confusion Matrix, Mean Squared Error (MSE), and Root Mean Squared Error (RMSE). I learned that the confusion matrix is mainly used for classification problems, where the goal is to categorize data into different classes. On the other hand, MSE and RMSE are used for regression problems, where the objective is to predict continuous values. Since my selected dataset was related to a regression problem, I applied MSE and RMSE to evaluate the performance of my model. This helped me understand how close my predicted values were to the actual values and how to measure the accuracy of my model effectively. Learning these evaluation techniques gave me a deeper insight into how machine learning models are assessed in real-world applications.

Evaluation was one of the most serious parts of the learning process because it showed that a model cannot be accepted only because it produces output. The output must be tested. If a model predicts values, I need to know how far those predictions are from the actual values. MSE and RMSE helped me measure that error. The confusion matrix gave me a separate understanding of classification problems, where correct and incorrect categories are compared. Even though my project was regression-based, learning both types of evaluation helped me see how different problems require different performance measures.

The evaluation stage also connected with what I learned about Dr. Bilal Ahmad's professional focus as an AI, ML, and DL model training expert. His preference for precise, high-value datasets, especially medical datasets where accuracy matters, helped me understand why evaluation metrics are not just formulas but a responsibility.

This point made evaluation feel more meaningful. In a classroom project, an inaccurate model may only reduce marks, but in real-world systems, especially in sensitive fields, poor evaluation can cause serious problems. If a medical model is trained or tested carelessly, the consequences can affect real people. Thinking about this helped me develop respect for the discipline of machine learning. Accuracy, error, and validation are not just academic words; they are part of responsible technical practice.

## Choosing the Right Dataset

Another important part of this phase was the dataset selection process. Our instructor provided multiple datasets and asked students to choose one based on a "first come, first serve" system. This created a sense of urgency and decision-making among students, as everyone wanted to select a dataset that was both interesting and manageable. I carefully reviewed the available options and selected a dataset that suited my understanding and project requirements. This experience taught me how to make quick yet effective decisions and understand the importance of choosing the right dataset for a machine learning project.

Dataset selection also taught me that a project begins before coding. The dataset decides many things: the problem type, the features, the cleaning effort, the model choice, and the evaluation metrics. If the dataset does not match the student's level or the project objective, the work becomes harder later. I had to think about whether the columns were understandable, whether the target variable was clear, and whether I could explain the problem properly. This made the selection process more than just choosing an interesting topic. It became a decision that shaped the whole project.

Overall, this stage of my learning journey significantly enhanced my practical skills. I not only learned how to visualize and analyze data but also understood how to evaluate model performance and select appropriate datasets. Despite minor challenges during the process, I remained focused on learning and improving my skills, which ultimately helped me grow as a more confident and capable student in the field of machine learning.

Recording this stage in my portfolio is useful because it shows how my understanding moved from simple coding toward complete project thinking. A visitor can see that I learned about visual analysis, evaluation responsibility, and dataset selection together.
